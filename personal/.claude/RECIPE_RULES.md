# 家庭食谱规则 / 记忆（沉淀自与用户的对话）

> 这是食谱系统的"宪法"。新增/修改食谱、跑 `generate_month.py` 前都应遵循。配置项见 `config/family.yaml`。

## 一、家庭与份量
- **4 人**：2 成人 + 2 儿童（**7 岁、5 岁男孩，正在长身体**）。
- 食谱 `servings: 4`，分量按"2 大 + 2 小"推断；两小孩合计约 1 个成人饭量，但**必须保证优质蛋白**（长身高）。

## 二、口味（很重要：孩子爱吃才有意义！）
- **不吃辣**；**少油少盐**。
- **优先孩子爱吃的口味**（营养均衡前提下）：酸甜、番茄味、芝士/奶香、照烧/微甜咸香、蜂蜜烤香、玉米/南瓜/紫薯自然甜、肉松/海苔鲜香。
- 做不健康但没意义——口味与营养并重。

## 三、烹饪方式约束（健康）
- **不用油锅油炸**（油炸不健康）。**例外**：炸葱油这类小型熬油可以。
- **空气炸锅常用、首选**——用它的少油"炸/烤"替代油炸（如脆皮、薯角、烤鸡翅、烤吐司）。
- **没有微波炉**——任何步骤不得依赖微波。
- 可用厨具：破壁机 / 空气炸锅 / 不粘煎锅 / 炖锅 / 玻璃锅 / 炒锅 / 电饭煲 / 蒸锅。
- 推荐做法：蒸、煮、炒、煎、炖、凉拌、破壁（米糊/豆浆/果蔬汁）、空气炸锅烘烤。

## 四、早餐（每天）
- **"三件套"硬要求**：每份早餐必须同时含 **主食 + 蛋 + 饮品**。
  - 主食：面 / 饭 / 馒头 / 饼 / 包 / 吐司 / 饭团 / 粥 等（如黄金馒头片、葱油拌面）。
  - 蛋：各种蛋、各种做法（水煮 / 煎 / 蒸蛋羹 / 厚蛋烧 / 荷包 / 卤蛋 / 溏心 / 鹌鹑蛋…）。
  - 饮品：牛奶 / 豆浆 / 米糊 / 果蔬汁 / 糊露 等。
- **一个月不重样**（早餐食谱池 ≥31，当前 34）。
- **每天换口味，不连吃同食材**：早餐用**反聚类**——避免与昨天共享食材（如黑芝麻糊连吃两天）。
  - 生成器实现：`pick_recipes(..., avoid_tags=last_breakfast_tags)`，与昨天共享 `ingredient_tags` 越少越优先。
  - ⚠️ 早餐**不做**食材聚类（与「六、食材经济」相反）——早餐图新鲜，正餐才聚类省食材。
- 晨间 ≤30 分钟；**前一晚可预备**（night_prep）。
- **儿童发育营养**：优质蛋白（蛋/奶/鱼/虾/牛肉）、钙（奶/黑芝麻/虾）、DHA（三文鱼/核桃）、粗粮膳食纤维。

## 五、节假日 / 周末正餐（午餐、晚餐）
- **「早吃饱 / 午吃好 / 晚吃少」总原则**：
  - **午餐 = 硬菜（午吃好）**：取「硬菜池」(`recipes/dinner/` 的排骨/牛肉/蒸鱼/滑鸡等)，主菜含荤 + 自带素菜/主食，再加 1 道**粗粮**凑丰盛（共约 3 道）。
  - **晚餐 = 简餐（晚吃少）**：取「简餐池」(`recipes/lunch/` 的面/拌饭/炒饭/馄饨等)，自带主食+蛋白+汤，清淡好消化，**不再加硬加菜**。
  - ⚠️ 注意目录名与用途**故意错位**：`recipes/lunch/` 里放的是简餐（给晚餐用），`recipes/dinner/` 里放的是硬菜（给午餐用）。生成器在 `generate_month_plan` 里做了交换映射，新增食谱时按「菜的丰盛程度」放对目录，不要按文件名直觉。
- 每餐 **2–3 道菜**（通常 2–3）。
- **荤素搭配**：主菜含荤（肉/鱼/鸡/虾），加菜配素。
- **粗细搭配**：细粮主食（米/面）+ 至少 1 道粗粮（玉米/红薯/紫薯/南瓜/山药/杂粮糙米）。
  - 生成器实现：午餐加菜 `prefer_category="粗粮"`，保证每个全餐日午餐有粗粮。
- 必有主食；汤可选。
- ⚠️ **同一天跨餐不撞主食**：早/午/晚（暑假再加快手午餐）的**主食 signature**（标题第一个 `+` 前的主菜名）当天不得重复——早餐和晚餐不能都是「番茄鸡蛋面」。
  - 生成器实现：晚餐/快手午餐选择时传 `exclude_sigs`，排除当天已出现的主食。
  - **质检必跑**：生成后跑 `python3 scripts/qc_plan.py --months 6,7,8`，撞车则退出码非 0。改生成逻辑后必须过此质检再推送。
- 工作日只吃早餐（午晚在外/不安排）。**例外见下「五·B 暑假工作日午餐」。**

## 五·B、暑假工作日快手午餐（lunch_quick）
- **触发条件**：日期落在学校假期内（`config/vacations-2026.yaml`，当前为暑假 7/1–8/31）**且**是工作日 → 当天**早餐之外再加一道快手午餐**。非假期工作日不变（仍只早餐）；周末/节假日走完整午餐（lunch），不加 quick。
- **人数**：**2 人（一大一小）= 1 成人 + 1 儿童**，`servings: 2`（区别于其他餐的 4 人）。
- **时间**：**前一晚备好（night_prep）**，**第二天中午操作 ≤30 分钟（noon_steps）**——腌好/切好/预约电饭煲，中午只剩煮/炒/焖/蒸的快手步骤。
- **其余要求同前**：不辣少油盐、儿童友好口味、不油锅油炸（空气炸锅可）、无微波；保证优质蛋白（蛋/肉/鱼虾/豆腐）。
- **食谱池**：`recipes/lunch_quick/*.yaml`（当前 23 道，按食材聚类+轮换排布；暑假单月约 23 个工作日，做到 30 天窗口内不重样）。
- **字段**：用 `noon_steps`（中午步骤）而非早餐的 `morning_steps`；可加 `serving_note` 说明份量。

## 六、食材经济（减少浪费）
- 蔬菜/肉通常 **500g 起买**（最小购买量）。
- **某天用不完的食材，优先安排接下来几天继续消耗**。
  - 生成器实现：`pick_recipes` / `pick_side` 按 `ingredient_tags` 做**跨天食材聚类**，相邻几天优先复用同类食材。

## 七、目录与生成
- `recipes/breakfast|lunch|dinner|side|special|lunch_quick/*.yaml`（special 为节日特供如粽子，不进日常轮换；lunch_quick 为暑假工作日快手午餐，见五·B）。
- 加菜按 `category` 分：素菜 / 汤 / 凉菜 / **粗粮**。
- 生成：`python3 scripts/generate_month.py --year YYYY --month M`（只写 ≥今天的每日卡片）。
- 推送：`bash scripts/run_daily.sh` 或对话里说"食谱"；`notify_daily.py` 按当天日期读 `plans/daily/*.md`（北京 18:00 前=今日、之后=明日）。
