# 李坤的汇报写作风格

## ⚠️ 关键教训（必须记忆）

- **找对文档再说话**：从不瞎猜用户写的在哪儿。先定位、读原文、再做判断
- **不要拿别人的内容对比用户风格**：部门报告(邓爽) ≠ 算法组报告(李坤)
- **不要套模板**：GIC汇报不固定结构，跟着内容走。3个bullet能说完就不要写3段

## 三个汇报场景的区分

### A. 周报（给高炳涛，每周三）
- **风格**：判断先行、带标记（🔑/🔴/🟡/🟢）、每项=**主题词**（Owner）：判断句+数据+行动
- **例子**："**🔑 慢速模式 — 本周关键突破**（瞿鑫宇）：链路已跑通——里程差异0.3m，横向差异<0.1m。Todo：规模化验证"
- **特点**：每条带Owner+成本数据+风险标记自然融入；末尾有风险表+依赖表+下周重点

### B. GIC双周会（给刘先明，经高炳涛审核，双周五）
- **风格**：极简+配图。**3-5个bullet解决一节，不写段落。每节一幅图。**
- **结构**：不固定，根据内容选（目标→现状→挑战、背景→优势→挑战、规划型等）
- **每节必有**：whiteboard(讨论概览) + GPT-Image-2图(数据可视化)
- **作者标签**：标题后 `<cite>李坤</cite>`
- **语言**："目标：Q2可以实现HIL集群1000Km/天的RC路线仿真里程" — 短句子，像在说话
- **典型写法**：
  ```
  # HIL闭环仿真（李坤）
  [whiteboard]
  - 目标：...
  - 现状：效率比最优1:2.5，当前两台台架，目标需至少5台
  - 当前资源规划：下周保证5台GPU投入使用；7月中旬完成45套GPU采购
  [GPT-Image-2图：效率对比/资源规划]
  ```

### C. 日会/核心日会
- 参会不写，但纪要中的讨论要点反映了李坤关注的方向
- 见 daily-sync 采集

## GPT-Image-2 在报告中的应用

**位置**：每节1张，跟在bullet列表后
**用途**：数据可视化（KPI对比、进度Dashboard、架构图）
**生成流程**：SoCheap API（key在 pipelines/media_key.txt）→ 存本地 → 手动贴图到飞书
**贴图标记**：`【贴图：<filename>.png】`
