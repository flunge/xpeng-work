# 车型泛化验证

## 目标
同批scenario仅变车型参数，控制变量对比。5阶段验证，6/18 Cloudsim交付。

## 里程碑历史

### 2026-06-11 — 5阶段已过4，结论正向
- camera mask优化版结论正向（部分指标温和改善）
- 4.1.2版本部分更好但不能下"全面优化"结论
- 3DGS改动已合入，Cloudsim 6/18交付
- 判定原则：优先看趋势一致性，其次看数值偏差
- 当前效率40min/PPU node，96卡集群多任务竞争

### 2026-06-10 — 5阶段验证进行中
- 方案价值已被验证：可显著提升多车型验证覆盖效率
- 模型优化方向正确，有实质收益但不够均匀
- E29和部分F01子项不稳定，暂不算"全车型稳态提升"

### 2026-06-08 — 车型泛化需求
- 因克劳森未上线，计划手动处理并反馈
- 15号后完全交给业务组

### 2026-05 — 验证方案设计
- 核心思路：A车型3DGS场景→改变外参→B/C车型原位渲染+Difix修图→开环评测
- 对比"原图 vs 不换车型渲染 vs 换车型渲染"三类结果
- 5阶段验证流程：方案验证→实现→迭代→Camera Mask优化→Cloudsim自动化
