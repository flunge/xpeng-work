# 闭环场景集建设推进

## 目标

将闭环场景集从"demo可看"推向"业务可用"——场景能复现、结果能评估、结论能解释，最终支撑发版gating。

Q2目标：10+类头部问题闭环场景集、4城1000+km RC路线，对外问题集复现率80%+、效率1:25。

## 全流程

```
业务组(邓爽) → 定义闭环优先级/交付量/毕业标准
    ↓
生产组(吴益民/云荟) → 数据源获取→3DGS建模→扶摇平台生产
    ↓
算法组(李坤) → 确认复现/不复现，反馈问题case
    ↓
评估组(夏志勋) → Metric验证、召回率/误报
    ↓
业务组 → Top Diff确认、实测KPI确认 → 毕业上线
```

## 当前机制（6/12起）

- **周二下午定期会**：同步计划变动、拉通任务需求与交付时间
- **双向清单**：算法组 → 业务组（交付内容/时间/接收确认）+ 业务组 → 算法组（需求/优先级/变更）
- **关键接口人**：刘开拓（组织）↔ 黄佰民（业务组参与人员确认）↔ 郑丽娜（清单核对）
- **Agent能力**：吕文杰探索用Agent帮助业务快速使用场景集（即使结果不够准）

## 核心风险

- 🟡 业务组计划变动频繁，模板调整未及时同步（如增加渲染结果查看要求）
- 🟡 闭环metric开发延迟（6/20验收），可能影响场景毕业节奏
- 🟡 生产组缩容导致数据损失（~25%），影响场景集供给
- 🟢 6/12后有了定期沟通机制，信息同步问题预计改善
