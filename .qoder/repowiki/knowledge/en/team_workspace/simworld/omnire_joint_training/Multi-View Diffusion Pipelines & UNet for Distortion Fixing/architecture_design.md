Three cooperating components built on top of `diffusers`:
- `mv_unet.py`: A fork of `diffusers.models.UNet2DConditionModel` that monkey-patches `BasicTransformerBlock.forward` (line 199) to reshape hidden states from `(b v) n d` into `b (v n) d`, run self-attention over the concatenated view dimension, then reshape back — enabling cross-view attention inside every transformer block. The class also re-implements `forward()` to pass through ControlNet/T2I-Adapter residual hooks.
- `pipeline_difix.py`: `DifixPipeline` extends `DiffusionPipeline` with LoRA/IP-Adapter/TextualInversion mixins; it replaces `encode_prompt` with `_encode_text_cached` backed by an in-memory `_text_embed_cache` keyed by prompt string or token tuple, and pre-primes per-camera prompts via `prime_camera_prompt_cache`. It composes VAE + CLIP text encoder + the custom `UNet2DConditionModel` from this package plus a diffusers scheduler.
- `pipeline_gaussian_render_fixer.py`: `GaussianRenderFixerPipeline` subclasses `StableDiffusionInstructPix2PixPipeline` and overrides `__call__` to perform triple classifier-free guidance (text / image / uncond) by chunking the UNet noise prediction three ways, targeting image-to-image rendering correction.

Dependency direction: pipelines depend on the local `mv_unet` module; `mv_unet` depends only on `diffusers` internals (no pipeline code). There is no shared base class between the two pipelines — they are independent entry points sharing the same UNet backbone.
