The module is the `sim_interface.BaseSimulator` implementation for Reconic scenes. `reconic_simulator.ReconicSimulator` owns lifecycle (config loading via OmegaConf, calibration from `transform.json`, pose interpolation, scenario JSON overrides) and exposes `render` / `render_multi_cam` / `render_hil` which build `ImageInfo`+`CameraInfo` through `get_novel_view_info` → `prepare_novel_view_render_data` and call into the loaded `gaussian` model.

Rendering strategy is decoupled behind an ABC (`render_strategy/render_strategy.RenderStrategy`) with concrete subclasses (`DefaultRenderStrategy`, `DifixRenderStrategy`, `DifixCarMaskAwareRenderStrategy`, `OriginalPngRenderStrategy`). A `RenderStrategyFactory` picks one at runtime by evaluating condition lambdas parsed from `render_config_manager.render_config.yaml`; the factory caches the chosen instance. The closed-loop entry points `closed_loop_api.fun` / `fun_one_frame` are the only public API consumed by the sim-interface: they resolve ego pose, dispatch to the selected strategy, optionally attach debug GT images, and invoke the simulator's duck-typed `apply_clipiqa_to_info` hook when `CLIPIQA_ENABLED=true`.

Supporting pieces: `cp_pose.compute_rig2anchor` converts world→anchor frames using KDTree lookups in CP mode; `render_split_region.select_region_masks` selects ground regions per trajectory segment; `clipiqa_helper.ClipIQAHelper` is lazily imported and wrapped so IQA can be toggled off; `simulator_constants` centralizes camera/label mappings and rigid class lists; `simulator_helpers` provides shared utils (numpy↔bytes, config path resolution, uint8 conversion). Dependency direction is one-way: closed_loop_api → RenderStrategyFactory → RenderStrategy → ReconicSimulator; the config manager is a singleton read-only source of truth for strategy/model selection.
