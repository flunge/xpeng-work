Two sibling sub-packages form a thin CLI layer over the rest of `reconic`:

- `cli/` — three independent `argparse` scripts, each with its own `main()` / `if __name__ == "__main__"` entry point: `train_cli.py` dispatches to one of `GenerativeReconTrainingLoop`, `XpengTrainingLoop`, or `ReconTrainingLoop` based on config keys (`joint_training_cfg`, `xpeng_trainer`); `eval_cli.py` loads a checkpoint via `import_str(cfg.recon_trainer.type)`, runs `do_evaluation`, and optionally spins up a viewer; `eval_metrics_cli.py` is a standalone post-processing pipeline that reads rendered PNGs from a result directory and computes detection metrics.
- `metrics/` — pure metric utilities with no CLI glue. `get_gt_infos.GroundTruthInfo` projects 3D instance corners (from `DrivingDataset.get_instance_infos`) into 2D bboxes under an optional camera Z-shift; `eval_detection.Evaluate2dDection` wraps an Ultralytics YOLO model, performs batched inference with resize+pad, matches predictions to GT via IoU, and accumulates per-class PR curves to compute AP/mAP; `metric_utils.py` holds shared helpers (bbox scaling/padding, FID via `torchmetrics.image.fid.FrechetInceptionDistance`, PSNR, drawing). The `__init__.py` is empty, so consumers import modules directly.

Dependency direction is strictly one-way: `cli/*` imports from `reconic.metrics.*` and from higher-level packages (`reconic.training_loop`, `reconic.utils.eval`, `reconic.datasets.driving_dataset`, `reconic.models.video_utils`); `metrics/*` only depends on `..datasets.*` and internal helpers, never on CLI code.
