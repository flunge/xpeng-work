Two parallel base implementations exist, selected at import time by the `GS_MODE` environment variable: `base_render.py` defines `BasicTrainer_render` (a pure `nn.Module` with no training loop) and `base.py` defines `BasicTrainer(BasicTrainer_render)` which adds the full training pipeline (`forward`, `backward`, `compute_losses`, `compute_metrics`, `preprocess_per_train_step`, `postprocess_per_train_step`, viewer integration). Concrete trainer subclasses are split by scene topology:
- `single.py::SingleTrainer` — one Background Gaussian set plus optional misc classes (Sky/Affine/CamPose), initialized from lidar/random points.
- `scene_graph.py::MultiTrainer` — composes multiple Gaussian classes (Background, RigidNodes, SMPLNodes, DeformableNodes, Ground, Trafficlight, RigidNodesLight) registered via a `model_config` dict; each class maps to a `GSModelType` enum value and is instantiated through `utils.misc.import_str`.

Shared rendering logic lives in `BasicTrainer.render_gaussians`, which dispatches between `gsplat.rendering.rasterization` and an alternative `xpeng_raster.rasterization` backend. Multi-camera batched rendering is provided by `collect_gaussians_of_multi_cams` + `multi_cam_render_gaussians`. The `__init__.py` re-exports either the render-only or full-trainer `BasicTrainer` depending on `GS_MODE=render`, so callers never need to know which variant they got. Dependency direction is strictly subclass → base → shared utils/models/datasets; trainers do not import each other except through the package-level switch.
