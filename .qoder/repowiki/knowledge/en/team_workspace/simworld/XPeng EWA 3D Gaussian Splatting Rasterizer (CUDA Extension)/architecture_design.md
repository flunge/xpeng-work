Built as a single `torch.utils.cpp_extension.CUDAExtension` named `xpeng_raster._C`, exposing four C++/CUDA entry points via pybind11: `projection_ewa_3dgs_fused_fwd_i`, `rasterize_to_pixels_3dgs_fwd`, `intersect_tile`, `intersect_offset`. The Python package `xpeng_raster/rendering.py` is the only public API; it wraps the extension through a `_LazyC` loader so the `.so` is not imported at module load time. Internally the C++ layer (`ext.cpp`) casts an integer camera-model token to `CameraModelType` before delegating to functions declared in `include/{Projection,Rasterization,Intersect,Cameras}.h`; the corresponding `.cu` implementations live under `cuda/csrc/` and are compiled with `-O3 -lineinfo` plus optional `-use_fast_math` controlled by the `XPR_FAST_MATH` env var. Build manifests: `setup.py` enumerates sources/includes and registers `BuildExtension`, while `pyproject.toml` pins `setuptools>=64`, `wheel`, `torch`, `pybind11` as build requirements. Three helper scripts under `scripts/` wrap the same `setup.py build_ext --inplace` invocation for different host/container environments.
