- Every public method of `BaseSimulator` that delegates to a concrete renderer raises `NotImplementedError`, making subclasses implement `init_parameters`, `init_models`, `setup_models`, `setup_feedforward_models`, `_label2camera`, `_camera2label`, and `get_camera`.
- Per-camera identity is always expressed as a pair of bidirectional dicts (`_label2camera`, `_camera2camera`) keyed by integer camera IDs and string names like `cam0`/`cam2`–`cam7`, and all loops iterate `self.cameras` then resolve the name through these maps.
- Redistortion is provided in symmetric CPU/GPU pairs (`redistort`/`redistort_gpu`, `distort_image_with_distortion_map`/`distort_image_with_distortion_map_gpu`) that share the same signature and both accept `(calib_info, cam_name, img, img_real, img_mask, distortion_maps)`.
- Optional heavy dependencies are imported lazily inside try/except blocks (e.g. `dynamic_assets.DCCF.scripts.camera_filter_smoother`) and fall back to `None` with a printed warning instead of failing at import time.
- Pose data is represented as 4×4 homogeneous numpy matrices built from quaternions via the local `quaternion_matrix` helper, and temporal queries use KDTree or Slerp rather than raw dict lookups.
