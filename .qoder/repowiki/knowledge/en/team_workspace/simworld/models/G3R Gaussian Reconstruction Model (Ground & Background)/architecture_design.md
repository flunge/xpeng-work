The module is organized as a self-contained PyTorch package under `models/g3r/`:
- `config.py` provides two region-specific configurations (`ground`, `bkgd`) via `obtain_config()` and an inference-time override in `modify_inference_config()`.
- `dataset.py` defines `XpengDataset`, which loads PLY surfels or background points, quantizes them to voxel grids via `torchsparse.utils.quantize.sparse_quantize`, and batches multi-view RGB + camera intrinsics/extrinsics per camera; `sparse_scenes_collate` stacks variable-length view sequences into batched tensors.
- `g3r_net.py` contains the core `G3RReconstructor` nn.Module: it initializes per-point priors (xyz, quat, color, scale, opacity), iteratively refines latent features over `T_iterations` using a cosine schedule `gammas[t]`, feeding concatenated `[S_t, grad_S_t]` through `SparseResUNet` from `sparse_unet.py`; a `NeuralGaussiansDecoder` maps updated features back to Gaussian attributes. Rendering uses `gsplat.rasterization` with antialiased mode and packed sparse gradients during training.
- `train_g3r.py` is the training entry point: loops over cases in `utils.training_cases`, downloads missing data via `downclip_utils.down_training_data`, runs the epoch loop, saves checkpoints every `evaluation_step`, and writes per-epoch loss/PSNR logs.
- `inference.py` implements `inference_g3r_interface()`: processes each camera independently, optionally averages Gaussians across cameras weighted by `camera_update_weight`, repairs unseen points, evaluates metrics, and exports `.ply` visualizations.
- `fuyao/` holds production deployment scripts (`deploy_g3r.bash`, `run_g3r_train.bash`) that submit jobs to the Fuyao cluster with a pinned Docker image.
- `utils/` groups shared helpers: `general_utils` (NetMode enum, schedules, I/O), `loss_utils` (PSNR), `math_utils` (quaternions, KDTree distances), `ply_utils` (Gaussian save/merge/repair), `downclip_utils` (data download), `training_cases` (case list).
Dependency direction is one-way: train/inference → g3r_net → sparse_unet + utils; dataset depends only on utils; config is consumed everywhere.
