Pure data directory of YAML configs consumed by the Street Gaussians training/evaluation pipeline; no executable code. Top-level subdirectories group configs by dataset or experiment campaign: `example/` (single-sequence Waymo baselines), `experiments_waymo/` (Waymo validation runs), `m1/` (XPeng M1 campaigns split by sequence length `10s` / `20s`, with further folders per git commit hash `c-<sha>` and a `-release` suffix for stable variants), `rs128/` (RS128 clip experiments), plus auxiliary campaigns like `fm_compare/`, `fm_fixed/`, `fm_oncall/`, `fm_vision/`, `fm_vslam/`, `subrun/`, `subrun_clips/`. Each config file is self-contained and declares a flat schema: top-level keys `task`, `source_path`, `exp_name`, `mode`, `loaded_iter`; nested sections `data:` (dataset type, frame/camera selection, colmap flags), `model.gaussian` + `model.nsg` (Gaussian Splatting hyperparameters and scene-part toggles), `train` / `train_xpeng` (iteration schedules, pruning/densification thresholds), `optim` (LR schedules, loss lambdas including ground/sky/mask/track terms), and `render` (FPS, output flags). The `task` field encodes the experiment identity (e.g. `waymo_full_exp`, `m1/<commit>`, `xpeng_<hash>`) used by the launcher to resolve defaults and logging paths.
