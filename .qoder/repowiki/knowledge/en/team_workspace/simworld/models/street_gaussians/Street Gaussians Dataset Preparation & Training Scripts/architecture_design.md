Flat script collection organized by dataset domain under `kitti/`, `waymo/`, and `xpeng/`, plus shared top-level helpers. Each domain provides a `colmap_<dataset>_full.py` entry point that reads scene metadata via `lib.config.cfg` and `lib.utils.<dataset>_utils`, splits frames with `lib.utils.data_utils.get_val_frames`, copies images into per-camera subfolders (`cam_{id}/frame.png`), writes COLMAP's `images.txt`/`cameras.txt`/`points3D.txt` (and a `cam_rigid_config.json` for multi-camera rigs in Waymo), then invokes the external `colmap` CLI (`feature_extractor`, `exhaustive_matcher`, `point_triangulator`, optionally `rig_bundle_adjuster`) through `os.system`. Per-dataset mask/depth generators live alongside: `generate_semantic_mask.py` / `generate_dynamic_mask.py` / `generate_sky_mask.py` (KITTI) and `generate_mono_depth.py` / `generate_lidar_depth.py` / `generate_mono_normal.py` / `generate_sky_mask.py` (Waymo). Top-level `convert.py` is a generic COLMAP feature extraction + undistortion + image pyramid builder driven by command-line flags. Shell wrappers `train_waymo_example.sh` / `train_waymo_exp.sh` / `render_waymo_example.sh` / `render_waymo_exp.sh` iterate over scenes and launch `python train.py --config configs/example/waymo_train_<scene>.yaml`. `test_gaussian_rasterization.py` is a standalone rasterizer sanity check. Dependency direction is one-way: scripts import from the sibling `lib/` package but never the reverse; they also depend on third-party CLIs (`colmap`, `magick`) rather than Python SfM libraries.
