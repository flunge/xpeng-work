Three-file leaf module with a clear dependency direction: `yacs.py` is an inlined copy of Facebook's YACS `CfgNode` (dict-like nested config with attribute access, merge-from-file/merge-from-list, freeze/deepcopy, deprecated/renamed key handling) — no external yacs install is required. `config.py` constructs a single global `cfg = CN()` root, declares every top-level namespace (`workspace`, `eval`, `train`, `train_xpeng`, `optim`, `model`, `data`, `render`, `fixer`, `viewer`) as nested `CN()` subnodes with hard-coded defaults, then uses `argparse` to accept `--config`, `--mode`, `--det`, `--local_rank`, `--print_cfg` plus REMAINDER opts; it also reads `/tmp/cfg_path_output.yaml` to dynamically override the YAML path before calling `make_cfg(cfg, args)` from `lib.utils.cfg_utils`. `globals.py` holds version and semantic-class mappings (`SEMANTIC_CLASSES`, `DATASET_CLASSES_IN_SEMANTIC`, `SemanticType` enum). `__init__.py` re-exports only `cfg` and `args` so callers import via `from lib.config import cfg, args` without seeing internals.
