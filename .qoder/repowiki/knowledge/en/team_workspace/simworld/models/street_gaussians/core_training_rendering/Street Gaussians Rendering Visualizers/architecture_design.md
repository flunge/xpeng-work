Three sibling classes with no shared base class: `BaseVisualizer` (legacy single-stream renderer), `StreetGaussianVisualizer` (core street-gaussian output including background/object decomposition plus normals), and `XpengVisualizer` (extended variant adding redistortion/harmonization pipelines and merged/eval video helpers). Each class owns its own frame buffers (`self.rgbs`, `self.depths`, `self.diffs`, …) and a `visualize(result, camera)` entry point driven by the training loop. Per-frame rendering is split into `visualize_diff` / `visualize_depth` / `visualize_normal` helpers; `summarize()` at the end of a sequence calls `save_video_from_frames`, which either writes one MP4 (single camera), concatenates cameras side-by-side when `cfg.render.concat_cameras` matches all observed cams, or falls back to one MP4 per camera. Output layout is cam-subfoldered under `rgb/`, `acc/`, `diff/`, `depth/`, `gt-rgb/`, etc., and image/video I/O is gated by `cfg.render.save_image` / `cfg.render.save_video` / `cfg.render.save_misc_images`. All three depend only on `lib.utils.camera_utils.Camera`, `lib.utils.img_utils.visualize_depth_numpy`, and `lib.config.cfg`; there is no inter-class coupling inside this package.
