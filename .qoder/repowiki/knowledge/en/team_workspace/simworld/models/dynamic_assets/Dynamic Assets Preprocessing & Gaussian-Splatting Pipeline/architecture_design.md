Standalone container image built from `Dockerfile` on `python:3.13.5`, installing system tools (`zip`, `curl`, `ca-certificates`, `zstd`) and Alibaba Cloud `ossutil` via its install script so assets can be pulled from OSS; Python deps are pinned in `requirements.txt` (numpy, matplotlib, pyyaml, pandas). The runtime entrypoint is not baked into the image — `CMD` is commented out — and users invoke scripts directly, notably `3D_real_car_train.py`, which orchestrates a two-stage workflow described in `README.MD`: (1) data preprocessing through a COLMAP/segmentation/clean/standard/rescale pipeline under `three_D_real_car_preprocess/three_D_RealCar_Toolkit/data_preprocess/resources/configs`, and (2) Gaussian Splatting training that converts point clouds into PLY files consumed by the simworld asset scene. External data lives at `/workspace/group_share/adc-sim/users/wangyl11/3dgs_dynamic/dataset/` and configs are mounted or copied into the container at runtime.
