Each `.py` file is a self-contained training script exposing `main(cfg)` that wires together three layers from the shared `iharm` package: model construction via `BMCONFIGS[...]` + `DeepImageHarmonizationUpsampleHSL_V3` or `HRNetIHModel`, dataset assembly through `ComposeDataset` / `ComposeDatasetUpsample` backed by `HDataset` / `HDatasetUpsample` on HFLICKR, HDAY2NIGHT, HCOCO, ADOBE5K, and trainer orchestration via `SimpleHTrainer` (256-only) or `UpsampleDCCFTrainer` (low-res→high-res upsample pipeline). The directory groups experiments by backbone (`hrnet18s_v2p_idih.py`) and DCCF variant (`issam_dccf_*`, `idih_dccf_*`), with suffixes encoding input resolution (`_HR`, `_LR`), color space (`_hsv1_clamp`, `_hsv_1_clamp`), and augmentation tweaks (`_sat_clamp`, `_upsample_hsl_refine_HR/LR`). Dependency direction is one-way: configs import from `iharm.*`; nothing in this module is imported back.
