The module is the upstream `graphdeco-inria/gaussian-splatting` repo embedded as a git submodule under `gaussian-splatting/`. It follows a four-component layout:
- CLI entry points at the root (`train.py`, `render.py`, `convert.py`, `full_eval.py`, `metrics.py`) parse arguments via `arguments/__init__.py`'s `ParamGroup` hierarchy (`ModelParams`, `PipelineParams`, `OptimizationParams`) and dispatch to shared runtime.
- `scene/` holds data loading (`dataset_readers.py`, `colmap_loader.py`, `cameras.py`) and the core `GaussianModel` class (`gaussian_model.py`) which owns all learnable tensors (`_xyz`, `_features_dc/rest`, `_scaling`, `_rotation`, `_opacity`, per-image `_exposure`) plus densification/pruning bookkeeping (`max_radii2D`, `xyz_gradient_accum`, `denom`).
- `gaussian_renderer/__init__.py` is the single Python-facing render API: it builds `GaussianRasterizationSettings`, calls into the CUDA rasterizer, optionally applies exposure compensation, and returns `{render, viewspace_points, visibility_filter, radii, depth}`.
- `submodules/` are three independently built PyTorch C++/CUDA extensions installed via their own `setup.py`: `diff-gaussian-rasterization` (forward/backward splatting + optional `SparseGaussianAdam`), `simple-knn` (GPU kNN for initial scale seeding), and `fused-ssim` (fast SSIM). Each ships its own `ext.cpp` + `.cu` sources and is imported lazily behind try/except guards so the code runs without GPU/CUDA if desired.
- `utils/` provides cross-cutting helpers (`sh_utils`, `graphics_utils`, `loss_utils`, `image_utils`, `camera_utils`, `general_utils`, `system_utils`, `dynamic_training_utils`) consumed by both training and rendering paths. The dependency direction is strictly one-way: CLIs → arguments → scene/gaussian_model → gaussian_renderer → CUDA submodules; utils are leaf helpers with no imports back into the top-level scripts.
