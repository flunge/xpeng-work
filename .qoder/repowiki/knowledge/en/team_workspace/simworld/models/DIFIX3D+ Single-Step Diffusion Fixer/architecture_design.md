The module is a self-contained fork of the upstream `nv-tlabs/Difix3D` repo, layered as:
- `src/` core: `model.py` defines the trainable `Difix` wrapper around a frozen `diffusers` StableDiffusion pipeline (SD-Turbo VAE + UNet + CLIP text encoder), adding LoRA on VAE skip connections and a one-step `DDPMScheduler`; `mv_unet.py` monkey-patches `BasicTransformerBlock.forward` with joint-view self-attention that can mask ref-view tokens via an additive attention bias; `pipeline_difix.py` is a custom `DiffusionPipeline` subclass exposing `from_pretrained("nvidia/difix[_ref]")`; `train_difix.py` orchestrates `accelerate` multi-GPU training with LPIPS/VGG losses and dual-resolution bucket sampling; `dataset.py`, `loss.py`, `utils_difix.py`, `config_train.py`, `inference_*.py`, `ref_perturb.py` are supporting utilities.
- Top-level `fixer.py` / `fixerTrt.py` expose a `DifixFixer` class that loads a pretrained checkpoint, optionally warms up torch.compile buckets, and provides `fix_image` / `fix_image_xpeng` methods used by the surrounding SimWorld codebase.
- `configs/*.yaml` enumerate experiment variants (v1–v6, origin/2buckets/gram/t100/1w variants) consumed by `config_train.py`.
- `preprocess/` scripts build the JSON dataset schema (`image`/`target_image`/`ref_image`/`prompt`) from raw data; `deploy/` wraps accelerate launch commands; `tools/` exports VAE to ONNX/TensorRT or torch.compile cache.
Dependency direction: `fixer.py` → `src.model.Difix` → `src.pipeline_difix.DifixPipeline` → diffusers transformers; training entry points depend on `src.*` only; configs and deploy scripts are pure data/shell.
