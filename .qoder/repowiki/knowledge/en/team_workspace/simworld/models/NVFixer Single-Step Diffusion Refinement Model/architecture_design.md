Standalone sub-project rooted at `src/` with three entry points: `train_nvfixer.py` (Accelerate DDP training loop), `inference_pretrained_model.py` (single-step denoising), and `evaluate_test_dataset.py` (PSNR/LPIPS evaluation). The core model is `pix2pix_turbo_nocond_cosmos_base_faster_tokenizer.py`, which instantiates the Cosmos 0.6B text-to-image pipeline via `get_cosmos_predict2_text2image_pipeline(model_size="0.6B", fast_tokenizer=True)` and wraps it as `Pix2Pix_Turbo`. Two lightweight adapters are appended to the frozen Cosmos backbone — `ReferenceTokenAdapter` (cross-attention over reference tokens) and `ReferenceDetailAdapter` (multi-scale residual detail map) — both optional behind flags like `use_ref_cross_attn` / `use_ref_detail_adapter`. `model.py` provides helper schedulers (`make_1step_sched*`) that pin a DPMSolverMultistepScheduler to 1 timestep, plus monkey-patched VAE encoder/decoder hooks for skip connections. Training losses combine L2, LPIPS, CLIP similarity, GAN, and Gram-style style loss; checkpointing uses a custom `save_ckpt`/`load_ckpt_from_state_dict` that serializes UNet + VAE + adapter state dicts plus optimizer/scheduler. `utils/` holds dataset collators (`PairedDatasetV2`, `CamGroupedBatchSampler`), eval helpers, and loss modules. Deployment is orchestrated through `deploy/*.bash` scripts that wrap `accelerate launch` with NCCL envs, while `fuyao_deploy_fixer/` provides cluster-specific wrappers. A required `tokenizer.patch` must be applied inside the `nvcr.io/nvidia/cosmos/cosmos-predict2-container:1.2` before training.
