Flat collection of independent scripts organized by task rather than a single package:
- `eval_tools/` — the core evaluation pipeline: `eval_fm_error.py` parses per-model `sim_*/*.json` files, matches real/sim frames via `source_time`, computes per-clip RMSE on acceleration/position/longitudinal/lateral axes, writes a CSV with an IQR-based outlier flag plus an AVERAGE row; `eval_main.py` orchestrates this flow and delegates plotting to `eval_plot.py`; `compare_inference_results.py` and `eval_tasks_download.py` are helpers.
- `dynamic_check/` — geometry helpers (`utils_geom.py`: quaternion→rotation, bbox corner projection, 2D mask generation) consumed by `visualize.py` for dynamic-object checks.
- `render_time_analysis/` — log download + timing analysis pair (`log_downloader.py`, `time_analyze.py`).
- `scripts/` — one-shot operational scripts: `deploy_render.bash` invokes the Fuyao scheduler to run `render_oss_mode.py`, which downloads trained Street-Gaussian tarballs from Alibaba Cloud OSS (`oss2`) and launches `street_gaussians/render_sim.py`; other scripts handle point-cloud counting, video merging, reconic log visualization, and IPython-time-collecting.
- `dockerfile/` — self-contained build images: `dds_parser`, `latest_a100`, `latest_ppu` each ship a minimal Dockerfile; `train_and_render_fm/Dockerfile` layers a base simengine image with CUDA 8.0/8.6/9.0 wheels, Colmap, ffmpeg, torchsparse, gsplat, diff-gaussian-rasterization, simple-knn, nvdiffrast, and a custom `requirements_new.txt`, then sets `tini` as PID 1.
Dependency direction is strictly outward: tools import only stdlib / numpy/pandas/cv2/oss2/yaml and sibling modules within their own subdirectory; nothing in this module is imported by the rest of `team/simworld`.
