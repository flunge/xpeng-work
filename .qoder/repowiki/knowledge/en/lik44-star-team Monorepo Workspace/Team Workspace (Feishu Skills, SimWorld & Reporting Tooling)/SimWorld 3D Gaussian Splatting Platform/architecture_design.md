The workspace is organized as a flat collection of independently packaged sub-projects rather than a monolithic Python package. Cross-child wiring follows three patterns:
- Shared CUDA primitive: `libs/xpeng_raster` provides the EWA Gaussian rasterizer that Street-Gaussians (`models/street_gaussians`) and Reconic (`omnire_joint_trainning`) both import.
- Simulation contract: `sim_interface/simulator_base.py` defines the abstract `BaseSimulator` API; concrete simulators in `models/street_gaussians/sim_bridge`, `omnire_joint_trainning/src/reconic/simulator`, and `xpeng_data_process` all implement it so the same UCP/Fuyao orchestration layer can drive different backends.
- Pipeline entry points: `pipeline/` (Fuyao batch jobs + UCP processors) and `agents/train-sim-eval-agent/` are the only consumers that wire together `xpeng_data_process` → `omnire_joint_trainning` → `models/*` → `tools/eval_tools`, while `skills/` exposes agent-facing SKILL.md guides to invoke these same scripts via `lark-cli`.
Each child ships its own `pyproject.toml` / `setup.py` / `requirements.txt` and is installed or run in isolation; there is no top-level `__init__.py` or shared runtime loader.
