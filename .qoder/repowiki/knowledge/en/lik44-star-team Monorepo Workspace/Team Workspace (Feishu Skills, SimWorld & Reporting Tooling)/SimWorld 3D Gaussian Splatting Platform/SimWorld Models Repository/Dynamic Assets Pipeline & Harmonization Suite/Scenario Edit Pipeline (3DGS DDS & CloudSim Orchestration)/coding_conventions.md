- Each script is self-contained with its own `logging.basicConfig(level=logging.INFO)` and `argparse` parser rather than sharing a logger or argument factory.
- External service calls (CloudSim REST, XPU copy, ossutil) are wrapped in a `@retry(retries=3, delay=2)` decorator so transient failures are automatically retried before propagating.
- DDS tables are addressed dynamically by suffixing `SimCameraImageTopicOnWebviz@<table_suffix>` discovered from the existing `CameraVideoTopic*` table name instead of hard-coding table names.
- Per-camera configuration (resolution, fps, bitrate, target image size) is centralized in the `CAMERA_CONFIG` dict keyed by logical camera ID, with sensor IDs remapped via `sensor_2_cam_type`.
- All HTTP requests carry an `X-Sign` header built from a fixed app_key/version/account/timestamp/signature tuple generated by `generate_hmac_sha256_signature`.
