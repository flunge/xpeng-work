The module is a thin wrapper around the diffusers StableDiffusionInstructPix2PixPipeline. `pipeline.py` defines `GaussianRenderFixerPipeline`, which subclasses `StableDiffusionInstructPix2PixPipeline` and rewrites `__call__` to concatenate input-image latents into the UNet channel dimension and apply dual classifier-free guidance over text, image, and unconditional branches — this is the core pix2pix conditioning loop. `train.py` provides `GaussianFixerTrainer`, which loads SD components (`DDPMScheduler`, `CLIPTokenizer`, `CLIPTextModel`, `AutoencoderKL`, `UNet2DConditionModel`) from a local base model directory, freezes VAE/text encoder, optionally injects PEFT `LoraConfig` on UNet attention modules (`to_k/to_q/to_v/to_out.0`), and runs an Accelerator-managed training loop with optional condition dropout (both prompt and image conditioning). `inference.py` exposes single-frame and batched entry points that load LoRA weights via `pipe.load_lora_weights` or `accelerator.load_state`, run the same pipeline, and compute FID against ground-truth frames. `utils.py` holds shared preprocessing: aspect-ratio-preserving resize-with-padding, mask loading/merging per camera ID, and the `preprocess_conditions` dataset transform that concatenates multiple condition images plus the edited target into one tensor and tokenizes `edit_prompt`. `train_origin_lora.py` is a frozen earlier variant without validation/FID/logging. `fuyao_run/*.sh` are orchestration scripts that iterate over pickled datasets and launch `train.py` / `run_infer.sh` / `gen_video.sh` / `search_param.sh`. Data-packaging helpers `pack_xpeng_train_data.py` / `pack_xpeng_render_data.py` build the pickle dicts consumed by trainer/inference.
