Two sibling scripts form the public surface of the Street Gaussians package:
- `train.py` bootstraps via `lib.config.cfg`, instantiates `Dataset` → `StreetGaussianModel` → `Scene` → `StreetGaussianRenderer`, then runs an iteration loop that composes L1/SSIM, sky, semantic, lidar-depth, normal, opacity-sparse, ground-flatten/symmetry/acc, color/pose correction, and box-reg losses before densify/prune and optimizer step; evaluation is delegated to `EvaluatorInTrainer` and TensorBoard.
- `render.py` reuses the same `Dataset`/`StreetGaussianModel`/`Scene`/`StreetGaussianRenderer` trio in `torch.no_grad()` mode and dispatches per-camera outputs through a pluggable `BaseVisualizer` (default `StreetGaussianVisualizer`) to write per-view images or a concatenated MP4 trajectory.
Both scripts share a single global `cfg` object from `lib.config` as the only cross-module contract — all hyperparameters, source paths, and output directories flow through it. The `lib/` subpackages (`config`, `datasets`, `models`, `utils`, `visualizers`) are pure libraries consumed by these two entry points; there is no shared base class between them beyond the config schema.
