Each top-level subdirectory (clip_iqa, difix, dynamic_assets, g3r, inspatio_world, nail_evolsplat, nvfixer, street_gaussians) is an autonomous Python project with its own requirements.txt, configs/, scripts/, and entry points; there is no shared package or cross-import between them. The only cross-cutting structure is the uniform layout convention: `configs/` for YAML/JSON experiment definitions, `src/` or flat `.py` modules for model + dataset + trainer, `deploy/` or `scripts/` for single/multi-GPU launchers, and a top-level `train_*.py` / `inference_*.py` driver. Dynamic assets sits one level above and orchestrates the others by invoking their CLI entry points (e.g. DCCF harmonization, 3DRealCar preprocessing, Gaussian Splatting training) rather than importing them as libraries.
