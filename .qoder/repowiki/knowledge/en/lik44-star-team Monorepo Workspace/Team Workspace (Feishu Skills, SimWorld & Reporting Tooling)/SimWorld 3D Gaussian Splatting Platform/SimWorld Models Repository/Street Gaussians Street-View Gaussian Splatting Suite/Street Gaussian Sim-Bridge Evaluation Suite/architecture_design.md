The module is organized around a single runtime core plus four evaluation entry points:
- `simulator.py` defines `StreetGaussianSimulator`, which extends `BaseSimulator` from `sim_interface.simulator_base`. It owns the full render pipeline: loading PLY checkpoints via `setup_models_ply`, building `Camera` objects with calibrated intrinsics (`_get_cam_intrinsic` / `_get_cam_intrinsic_noncrop`), rendering through `StreetGaussianRenderer`, redistorting to sensor space via `redistort_gpu`, and optionally running Difix image correction. Pose handling includes ego-pose smoothing (windowed pitch averaging) and CP-simulation mode that refines z using a ground-plane KNN fit.
- `evaluator_metric.py` provides `ImageEvaluator` + `EvaluatorMethod`: it iterates over stored timestamps/cameras, calls `simulator.render`, compares rendered vs. real images on disk under `cfg.source_path/images_origin/<cam>/<timestamp>.png`, and writes per-camera PSNR CSVs via pandas describe summaries.
- `novel_eval.py` subclasses `StreetGaussianSimulator` into `NovelEvaluator` (longitudinal + lateral-shifted viewpoints generated by `sim_interface.utils.get_longitudinal_interpolated_egoposes` / `get_lateral_shifted_egoposes`) and `NovelViewGenerator` (poses sourced from an external DDS JSON via `get_mflocalpose_from_dds_json`). Both write frames through `XpengVisualizer` and `NovelEvaluator.evaluate` computes FID against real images using `torch_fidelity.calculate_metrics`, saving results as `evaluation_results.json`.
- `extract_cp_pose_3dgs.py` is a standalone CLI tool that greps `sim_engine_out.log` for `3DGS] pose` lines, parses camera name/pose/time, and emits `local_pose.json` consumed by `NovelViewGenerator`.
- `eval_summary.py` orchestrates cross-checkpoint aggregation: downloads `psnr.csv` and `evaluation_results.json` from two model IDs (256, 1347) per clip listed in `eval_scenarios/eval_dataset_v2.txt`, concatenates them, and produces `psnr_summary.csv`.

Dependency direction is one-way: all evaluators depend on `simulator.StreetGaussianSimulator`; nothing in the module imports back into the evaluator scripts. External dependencies are pulled in by mutating `sys.path` at import time to reach sibling packages (`lib.*`, `sim_interface.*`).
