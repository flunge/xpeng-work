The package is organized into five top-level sub-packages that form a classic deep-learning stack:
- `model/` defines the neural architectures. `base/` holds the core harmonization backbones (`DeepImageHarmonization`, `SSAMImageHarmonization`, `DeepImageHarmonizationUpsampleHSL_V3`) plus shared building blocks in `modeling/` (ResNet, HRNet-OCR, DeepLabV3+, TransUnet, UNet). `backboned/` wraps these backbones into IH-specific modules (e.g. `HRNetIHModel`, `DeepLabIHModel`). `syncbn/` ships a custom CUDA sync-batchnorm extension under `modules/functional/csrc/`. Losses, metrics, modifiers, GAN helpers, and PS-domain filters live at `model/` root.
- `mconfigs/` provides factory dicts (`BMCONFIGS`) mapping string names to `(model_cls, params)` pairs; `backboned.py` augments `base.py` with backboned variants. This is the single point where model selection is configured.
- `data/` exposes PyTorch Dataset classes (`HDataset`, `HDatasetUpsample`) over a directory layout of `real_images/`, `composite_images/`, `masks/`, with transforms and composition utilities in sibling files.
- `engine/` contains trainer abstractions: `simple_trainer.SimpleHTrainer` is the default loop (train/val, TensorBoard via `SummaryWriterAvg`, checkpointing, optional DataParallel), while specialized trainers (`upsample_hsl_mc_trainer`, `upsample_dccf_backbone_trainer`, `balanced_dp`, `event_loop`, `mhead_trainer`) extend it for multi-head / interactive / DCCF-specific flows. Optimizer selection is delegated to `optimizer.get_optimizer`.
- `inference/` offers standalone prediction entry points (`predictor`, `predictor_upsample_hsl[_nobackbone]`, `predictor_upsample_interactive`) plus evaluation/metrics and input transforms.
- `utils/` bundles experiment logging (`exp.py`), a `logger` wrapper with TQDM sink, and misc helpers (`save_checkpoint`, `load_weights`).

Dependency direction is one-way: `engine` → `model` + `data`; `inference` → `model` + `data`; `mconfigs` → `model`; nothing inside `model/` imports from `engine` or `inference`. The `syncbn` CUDA extension is compiled separately (C++/CUDA sources under `csrc/cuda/`) and exposed through `modules.functional.syncbn`.
