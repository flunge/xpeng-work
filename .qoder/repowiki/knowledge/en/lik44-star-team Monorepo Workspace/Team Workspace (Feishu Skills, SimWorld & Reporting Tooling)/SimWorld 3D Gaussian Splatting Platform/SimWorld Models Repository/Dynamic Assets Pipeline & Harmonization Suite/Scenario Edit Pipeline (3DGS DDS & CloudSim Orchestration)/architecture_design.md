Three cooperating CLI scripts with one-way dependencies: `scenario_info_update.py` is the top-level orchestrator — it queries/duplicates a CloudSim scenario via an HMAC-SHA256 signed REST API (`/simulation/scenario/query/`, `/simulation/scenario/duplicate_scenario/`) and triggers an XPU Ceph copy through a separate pipeline endpoint. `image_3dgs_to_dds.py` is the heavy worker: it opens a DDS `.dat` file as SQLite, reads `CameraVideoTopic*` rows via `MsgBinder.msg.camera_service.CameraVideo/CameraImage` protobufs, synchronizes multi-camera timestamps within a 20 ms window, matches local PNG frames by filename timestamp, encodes each frame to HEVC Annex B with ffmpeg/libx265, and writes back the patched CDR payload plus optional raw image data into both the video and a derived `SimCameraImageTopicOnWebviz@<suffix>` table. It delegates cloud I/O to `dds_downloader.py`, which shells out to `ossutil` against the Wulanchabu endpoint. `dds_downloader.py` has no internal dependencies and is only imported by the encoder. The module exposes `process_one_dds(dat_path, base_dir, mode, discovery_file_path)` as the single entry point for the encode/fm_origin/img_test/origin modes; there is no package `__init__.py` or shared library boundary — callers import modules directly.
