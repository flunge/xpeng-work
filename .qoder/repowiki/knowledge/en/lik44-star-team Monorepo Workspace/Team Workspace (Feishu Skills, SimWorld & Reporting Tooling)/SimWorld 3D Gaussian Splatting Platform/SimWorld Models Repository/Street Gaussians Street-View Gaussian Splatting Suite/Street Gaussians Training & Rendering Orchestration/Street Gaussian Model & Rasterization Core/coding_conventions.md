- Each Gaussian primitive stores its state in private `nn.Parameter` attributes prefixed with underscore (`_xyz`, `_features_dc`, `_features_rest`, `_scaling`, `_rotation`, `_opacity`, `_semantic`) and exposes them through read-only `@property` getters that apply activations (exp, sigmoid, normalize).
- Per-primitive optimizers are registered as named param groups (`{'params': [...], 'name': <tag>}`) so that `update_optimizer`/`reset_optimizer`/`prune_optimizer`/`cat_optimizer` can target individual tensors by name.
- State persistence uses paired `state_dict()` / `load_state_dict()` methods that explicitly enumerate every field (including optimizer states when `is_final=False`) rather than relying on `super().state_dict()`.
- Dynamic composition is driven by string prefixes (`background`, `ground`, `obj_{id:09d}`, `sky`) stored in a `bidict` and accessed via `getattr(self, model_name)` after `set_visibility(include_list)` filters which sub-models participate in the current frame.
- Rendering kernels return a uniform dict contract `{rgb, acc, depth, viewspace_points, visibility_filter, radii}` plus optional feature keys (`normals`, `semantic`) split out by `feature_dims`, regardless of whether the backend is diff-gaussian-rasterization or gsplat.
