- `simulator_base.py` declares the abstract `BaseSimulator` class that every simulator implementation must subclass; subclasses override `init_parameters`, `init_models`, `setup_models`, `setup_feedforward_models`, `_label2camera`, `_camera2label`, and `get_camera`. The base class owns the common lifecycle: loading `calib.json`, building per-camera distortion maps, loading per-vehicle render masks from `assets/Vehicle_Mask_Render/<model_id>/`, converting anchor poses to transfer poses via DDS JSON, and exposing three rendering entry points — `simulate_one_frame`, `simulate_one_frame_batch`, and `simulate_one_frame_stream` (the latter uses two-stage CUDA streams for parallel render + redistort).
- `utils.py` is a pure-function library with no class state: it holds the canonical `_camera2label` / `_label2camera` mappings, implements `TransferposeIndex` (KDTree over translation vectors for nearest-anchor lookup), pose interpolation (`interpolate_pose_with_times`, `lookup_pose`, Slerp-based helpers), and paired CPU/GPU redistortion pipelines (`redistort`/`redistort_gpu`) that apply precomputed OpenCV remap grids and optionally blend masked regions back from real images.
- `visualizers/xpeng_visualizer.py` is a thin recording/writing helper (`XpengVisualizer`) that buffers frames per camera and writes merged MP4s via `imageio`; it depends on a module-level `cfg.render.fps` rather than taking fps as an argument.
- `assets/Vehicle_Mask_Render/` is a data-only directory: one subfolder per vehicle model ID, each containing per-camera `_camN.png` mask and `rgb_camN.png` reference image consumed by `load_render_mask`.
- Dependency direction is one-way: concrete simulators in sibling packages import this module's `BaseSimulator` and `utils`; this module imports only third-party libs (`torch`, `cv2`, `scipy`, `shapely`) and external project packages (`reconic.*`, `dynamic_assets.DCCF`) via optional lazy imports.
