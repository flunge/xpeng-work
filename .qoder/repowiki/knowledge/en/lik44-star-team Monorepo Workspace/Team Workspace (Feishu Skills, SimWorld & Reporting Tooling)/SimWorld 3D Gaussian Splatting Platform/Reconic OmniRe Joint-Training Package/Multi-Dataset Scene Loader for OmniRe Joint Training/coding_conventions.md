- Each dataset sub-package mirrors the same four-file shape: `<name>_sourceloader.py` (implements `ScenePixelSource` + `CameraData` subclasses), `<name>_human_utils.py` (instance/human parsing helpers), `<name>_preprocess.py` (offline conversion scripts), and optionally `<name>_preview.py`.
- Abstract base classes expose `@abc.abstractmethod` hooks (`load_calibrations`, `create_all_filelist`, `load_lidar`, `load_cameras`, `load_objects`) that every dataset subclass must override, while providing shared defaults like `to(device)`, `get_aabb()`, and normalized-time registration.
- Runtime class resolution uses `import_str(cfg.type)` against the fully-qualified class path stored in the OmegaConf config, keeping dataset wiring declarative rather than import-heavy.
- Per-scene metadata is read from a `metadata.json` at the scene root (vehicle model, camera names) and combined with `DATASETS_CONFIG` to derive per-camera load sizes and coordinate transforms.
- Instance/object taxonomy is centralized through `ModelType` (RigidNodes/SMPLNodes/DeformableNodes) and `ObjectType` (Vehicle/Pedestrian/Cyclist/Motorcycle/Other) IntEnums, with dataset-specific class-to-node mappings defined as module-level dicts.
