- `feishu-agent` is the inbound entry point: it exposes FastAPI HTTP endpoints for Feishu webhooks and a local agent API, parses messages into typed tasks, dispatches them via an executor registry, and persists state in SQLite.
- `train-sim-eval-agent` (`tse`) is the heavy-lifting worker: it builds simulation binaries (locally or via nested SSH→docker), submits jobs to the cloud sim platform, monitors execution, runs FM/PSNR evaluation using simworld `tools/`, and posts results back to Feishu.
- Cross-child wiring: `tse/integrations/feishu.py` implements its own Feishu client (tenant_access_token → image/file/message APIs) so both agents can send reports without sharing code; `tse/config.py` centralises all external endpoints (Temporal, sim platform, Feishu, eval output root) behind a pydantic-settings `Settings` object consumed by every activity/workflow.
- Shared runtime contract: both agents are Python services configured via `.env` / `pydantic_settings`, each with its own virtual environment and startup scripts under `scripts/`; they communicate indirectly through the sim platform and Feishu rather than direct RPC.
