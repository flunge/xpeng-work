Two parallel orchestration backends share a common `configs/` directory of YAML experiment definitions (`sim3dgs_v41x*.yaml`, `*_preprocess.yaml`, `error_definitions.yaml`) consumed by both paths:

- `fuyao/`: Bash wrappers (`deploy_preproc.bash`, `deploy_render.bash`, `deploy_reconic.sh`, `deploy_render_switch.bash`) invoke the `fuyao deploy` CLI to launch GPU jobs on the Fuyao cluster. Each sub-task has its own Python driver (`download_model_and_render.py`, `download_data_and_train.py`, `download_and_car_switch.py`) plus a thin `render.sh` / `run_preproc.bash` / `run_reconic.sh` launcher.
- `ucp/`: Implements the IPS/UCP processor contract with three cross-process hooks — `pre_processor`, `gpu_processor`, `post_processor` in `ucp_multi_vehicle.py`. State is persisted between processors as a JSON file (`pipeline_record.json`) under each scenario's `root_path`. An adaptor layer (`adaptors/multi_vehicle_adaptor.py`) translates platform-level `scenario_list`/`target_vehicle` into per-scenario data chunks for distributed processing. Supporting utilities cover DDS merging/slicing (`merge_slice_hil_dds_lib.py`), image/Difix preprocessing (`ucp_img_preprocess.py`, `ucp_difix_preprocess.py`, `ucp_difix_augment.py`), OSS I/O (`upload2oss.py`, `download_file_from_oss2.py`), quality checks, and error classification (`pipeline_error_codes.py`).

Dependency direction: `ucp_multi_vehicle.py` imports shared helpers from sibling files and from the parent `reconic` package (installed elsewhere in the repo); it does not import `fuyao/` code. Both backends depend only on the static YAML configs in `configs/`.
