- Each Fuyao task exposes a paired `deploy_*.bash` wrapper that resolves `repo_root` relative to its own location and forwards positional args to a sibling `run_*.bash` script, keeping cluster invocation uniform across preprocess/render/train/switch tasks.
- UCP processors follow a strict try/except pattern: on failure they call `send_kafka_error_message(record, ...)` then `clean_up_tmp_data(record)` before re-raising, ensuring error reporting and temp cleanup are always executed.
- Cross-processor state is never passed in-memory; instead `persist_record()` serializes the mutable `record` dict to `pipeline_record.json` under `root_path`, and subsequent processors load it via `load_record(root_path)`.
- Error classification is centralized: callers wrap exceptions through `classify_error(exc)` from `pipeline_error_codes.py`, which matches against keyword patterns loaded from `configs/error_definitions.yaml` and emits a structured JSON payload via `ClassifiedPipelineError.str()`.
- Experiment variants are expressed purely as YAML files under `configs/` (e.g. `sim3dgs_v416.yaml`, `sim3dgs_v416_preprocess.yaml`, `sim3dgs_v416_preprocess_pro.yaml`) without code changes, so adding a new version means copying and editing a config rather than touching Python.
