Two sibling sub-packages with a one-way dependency: `settings/` is consumed by `utils/`, never the reverse.

- `settings/yacs.py` is an in-tree vendored copy of Facebook's YACS (`CfgNode`) providing dict-like nested config nodes with attribute access, immutable/frozen mode, YAML + Python-source loading, recursive merge, deprecated/rename key tracking, and type coercion — used as the single source of truth for all runtime options.
- `settings/config.py` builds the default `CfgNode` tree via `make_default_settings()` (camera/lidar lists, per-step processor toggles under `steps_controller.*`, voxel sizes, projection params, XData cache paths, trigger-time offset maps), then merges user YAML overrides through `make_cfg` / `get_config_list`; `make_case_specific_settings` derives `clip_path` from `clip_id` or `subrun_list` and prunes lidar when `steps_controller.source == 'vision'`. `settings/globals.py` exposes only constants: `VERSION`, `SEMANTIC_CLASSES`, `DATASET_CLASSES_IN_SEMANTIC`, and the `SemanticType` IntEnum used across utils.
- `utils/` is a flat collection of ~20 independent helper modules grouped by domain: geometry/projection (`projection.py`, `general_utils.py`), COLMAP I/O (`colmap_parser.py`, `colmap_utils.py`), point-cloud/PLY/PCD I/O (`file_utils.py`, `ground_utils.py`, `static_recon_utils.py`), depth visualization (`depth_visualizer.py`, `depth_conf_*.py`), annotation/mask handling (`annotation_sf.py`, `annotation_utils.py`, `sam3d_utils.py`), sensor-specific adapters (`fuyao_utils.py`, `lomm_utils.py`, `novel_utils.py`), OSS/file orchestration (`oss_utils.py`, `file_utils.py`), and CLI/data-fetching helpers (`get_jira_clip.py`, `match_pose_and_cam.py`, `qa_pose.py`). Each file is self-contained; there is no central dispatcher — callers import exactly what they need.

Dependency direction: `utils/*` → `settings.globals` (semantic enums) and optionally `settings.yacs.CfgNode`; nothing in `settings/` imports from `utils/`.
