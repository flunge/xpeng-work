The module is split into two independent sub-packages sharing the same input domain (multi-view driving imagery with camera poses):

- `rogs/` implements RoGS (Road Gaussian Splatting). Entry point `train.py` loads a YAML config via `argparse`, builds an `XpengDataset`, constructs a `Road` mesh (`models/road.py`) by gridding the ego trajectory and cutting to a bounding box, then initializes a `GaussianModel2D` (`models/gaussian_model.py`) whose parameters are `_xy`, `_z`, `_scaling_xy`, `_rotation`, `_opacity`, `_label` plus either raw `_rgb` or spherical-harmonic `features_dc/rest`. Rendering goes through `diff_gaussian_rasterization_depthalpha`'s `PerspectiveCamera`; BEV rendering uses an `OrthographicCamera`. Losses (`L1MaskedLoss`, `CELossWithMask`, depth/smooth/z losses) are composed in `train.py`, with optional `ExposureModel` and `RoGSAffineModule` applied per-image. Densification/pruning is driven by gradient accumulation on the xy-plane.
- `rome/` implements RoMe (differentiable mesh reconstruction). `train_lightning.py` launches a PyTorch-Lightning `Trainer` over a custom `RomeNet` (`rome_net.py`) which selects a `SquareFlatGrid*` voxel grid from `models/voxel.py` based on which of `{vertices_rgb, vertices_label, vertices_z}` have non-zero learning rates, and optimizes extrinsics via `pose_model.ExtrinsicModel`. Rendering uses a custom `NvdiffRenderer` (`utility/nvdiff_renderer.py`). Outputs include per-epoch BEV RGB/semantic/depth, optimized camera extrinsics, and packed GTA-format meshes.

Both packages follow a data → model → render → loss → optimizer loop, share loss utilities (`loss.py`), and write results under a configurable output root. There is no cross-import between `rogs` and `rome`.
