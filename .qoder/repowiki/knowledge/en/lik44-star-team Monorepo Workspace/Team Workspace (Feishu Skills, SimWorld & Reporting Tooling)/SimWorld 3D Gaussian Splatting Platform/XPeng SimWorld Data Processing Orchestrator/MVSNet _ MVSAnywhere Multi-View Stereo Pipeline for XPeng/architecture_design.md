Two-layer layout under this scope:

- `mvsa/` — a self-contained Python package (installable via `setup.py` / `pyproject.toml`) that is the core MVS engine. Its entry points are `src/mvsanywhere/train.py` (PyTorch Lightning DDP training loop) and `src/mvsanywhere/test.py` (inference + optional TSDF fusion). All experiment knobs flow through `options.Options` dataclasses parsed by `OptionsHandler`, which merges YAML configs (`configs/base/*.yaml`, `configs/models/*.yaml`, `configs/data/xpeng_dense.yaml`, `configs/run/xpeng_run_config.yaml`) with CLI flags in strict precedence order.
- Internal layering inside `src/mvsanywhere/`: `datasets/` provides a `GenericMVSDataset` base plus dataset-specific subclasses (`xpeng_dataset.py`, `generic_mvs_dataset_official.py`); `modules/` holds the neural components (cost volume, feature volume, view-agnostic volumes, ViT blocks, UNet-style networks); `experiment_modules/` wraps models as Lightning modules (`rmvd_mvsa.py`, `sr_depth_model.py`); `tools/` bundles offline utilities including a custom C++/CUDA marching-cubes extension (`marching_cubes/ext.cpp`, `marching_cubes.cu`), TSDF fuser, partial fuser, point-cloud fuser, keyframe buffer, tuple generator, and mesh renderer; `utils/` is shared helpers (geometry, augmentation, metrics, rendering, visualization).
- `scripts/` sits at the top of `mvsa/` and contains one-off data preparation scripts (`data_scripts/generate_train_tuples.py`, `generate_blendedmvg_tuples.py`, `undistort_nerfstudio_data.py`, …), evaluation harnesses (`evals/mesh_eval.py`, `evaluation.py`), and inference helpers (`render_meshes.py`, `create_visibility_volume.py`).
- `utils/generate_data.py` (outside `mvsa/`) is an XPeng-specific data assembler: it ingests raw vehicle clips, groups them into trips, rewrites camera extrinsics per vehicle model, and produces the scan/tuple structure consumed by `XPengDataset`.

Dependency direction is strictly inward: `train.py` / `test.py` → `experiment_modules` → `modules` → `utils`; datasets depend only on `utils` and the generic base class. The XPeng adaptation lives entirely in `datasets/xpeng_dataset.py` plus its run config, leaving the rest of the MVS stack unchanged.
