Two entry points: `train.py` (Lightning Trainer bootstrap with DDP/DataParallel, GPU auto-selection via `pycg.exp`, checkpointing, TensorBoard/W&B logging) and `gsm_inference.py` (single-process evaluation that loads a checkpoint and renders `.splat` + PLY outputs). The core package `scube/` is layered as:
- `models/base_model.py` — abstract `BaseModel(pl.LightningModule)` providing OOM-safe train/val step wrappers, gradient clipping/nan-skip, hparams/TB metric registration, dataset loader factories (`train_dataloader`/`val_dataloader`/`test_dataloader`) that delegate to `scube.data.build_dataset`, and a `load_module` factory that dynamically imports `scube.models.<name>.Model` from a YAML spec.
- `models/gsm.py` — concrete GSM model composing `UnifiedEncoder` → `Pure3DUnet` backbone → `RGBRenderer`/`FeatureRenderer`, plus an optional skybox module; implements `train_val_step` driving `UnifiedLoss`.
- `modules/` — pluggable sub-packages: `encoders/` (LSS/Lift3D/point/map/box3d), `gsm_modules/{encoder,backbone,renderer,loss}` (config-driven via `hparams_handler` + `eval(target)`), `render/gsplat_renderer.py`, `sky/` (panorama/null skyboxes).
- `data/base.py` — `DatasetSpec` enum of all batch keys (point clouds, images, maps, boxes, semantics) plus `RandomSafeDataset` deterministic RNG wrapper and `list_collate` supporting `fvdb.GridBatch`/`JaggedTensor`.
- `utils/` — shared helpers (voxel ops, depth/mesh/color utilities, wandb, WDS, loss meters).
- `ext/common/` — JIT-compiled CUDA/C++ extensions (kdtree, knn, cutil_math) loaded through `ext/__init__.py`'s `load_torch_extension` helper.
- `configs/` — OmegaConf YAML hierarchy: `default/param.yaml` (global defaults), `train/gsm/*.yaml` (GSM-specific hyperparams including gsplat rasterize mode, alpha/depth weights, optimizer schedule), `dataset.yaml`.
Dependency direction is strictly inward: scripts → models → modules → utils/data; configs are consumed at runtime via `exp.ArgumentParserX(base_config_path=...)` rather than import-time constants.
