Two independent sub-languages coexist under one directory:
- Python image generators (`gpt_image_gen/`, `grok_image_gen/`) are standalone CLIs using only stdlib (`urllib.request`, `argparse`). Each script POSTs a job to `https://socheap.ai/media/image/generations` with model `gpt-image-2` or `grok-imagine-image`, polls `/media/image/generations/{id}` until `completed`/`failed`/`cancelled`, then downloads PNGs via `urlretrieve`. Secrets come from sibling `media_key.txt` (Bearer token regex fallback). `gen_batch.py` adds a simple in-process loop over a JSON spec file.
- JavaScript report workflows (`gic-report.js`, `weekly-report.js`) are Deno-style pipeline scripts exporting a `meta` object with named phases. They orchestrate data collection, analysis, synthesis, and output by invoking an external `agent(...)` function (provided at runtime by the Claude agent harness), calling `lark-cli` subprocesses against Feishu docs/IM, and optionally triggering image generation through SoCheap or the local image scripts. The two JS files share a secret-loading pattern: prefer `MEDIA_API_KEY` env var, fall back to `pipelines/media_key.txt`.
Dependency direction is flat — no shared library; each script is self-contained and invoked directly from the repo root or from higher-level orchestrators.
