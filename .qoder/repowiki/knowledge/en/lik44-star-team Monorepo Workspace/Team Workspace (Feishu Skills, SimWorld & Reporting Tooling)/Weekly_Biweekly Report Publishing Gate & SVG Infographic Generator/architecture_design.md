Flat collection of independent CLI scripts plus two PreToolUse/PostToolUse hooks, orchestrated by `preflight.py` as the single entry point that chains sub-checks:
- Content validation: `check_report.py` calls `lark-cli docs +fetch --doc-format markdown --format json`, then runs regex suites (EXCLUDE, AUDIENCE_BAD, SOURCE_LEAK, JARGON, EXP_CODE, HEARSAY, DUP_PHRASES) over the Markdown to emit a typed issue list; supports `--audience boss|xianming`, `--wartable`, `--prev`.
- Rendering gate: `gen_svg_infographic.py` is a pure-Python SVG generator (no satori/HTML) that hand-computes coordinates for cards/KPI/bullets, registers text bounding boxes into `_ALLBOXES`, and exits non-zero when `_check_overflow` detects any bullet spilling outside its card — consumed by preflight's "渲染闸".
- HTML→SVG bridge: `html2svg.mjs` uses `satori` + `satori-html` with a bundled CJK font (`fonts/NotoSansCJKsc-Bold.ttf`) extracted by `extract_font.py` from system NotoSansCJK-Bold.ttc via `fontTools.ttLib.TTCollection`; used only for legacy HTML snippets.
- Hooks in `hooks/`: `asr_guard.py` (PreToolUse) intercepts `lark-cli docs +create/+update` stdin payloads, scans for ASR-misheard acronyms (HIL/SIL/FP8), and exits 2 to feed stderr back to the model; `docx_write_verify.py` (PostToolUse) warns after risky block mutations (`block_replace`/`insert_after`/`delete`) because lark-cli returns ok:true on stale ids.
- Dependency direction is one-way: `preflight.py` → `check_report.py` + `gen_svg_infographic.py`; hooks are invoked externally by the harness around `lark-cli` calls; no shared Python package — each script is self-contained and callable standalone.
