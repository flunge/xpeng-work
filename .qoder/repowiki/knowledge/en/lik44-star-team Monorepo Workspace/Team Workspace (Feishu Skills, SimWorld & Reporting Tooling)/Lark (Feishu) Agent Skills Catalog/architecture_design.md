Each subdirectory under `team/.agents/skills/` is a self-contained skill package for one Feishu product domain (e.g. `lark-im`, `lark-slides`, `lark-base`). A skill's entry point is its top-level `SKILL.md`, which declares a YAML front-matter (`name`, `version`, `description`, `metadata.requires.bins: ["lark-cli"]`, `metadata.cliHelp`) so the agent runtime can discover it and know which CLI binary to invoke. Product-specific reference material lives in a sibling `references/` directory (one `.md` per shortcut or DSL), while richer skills add `assets/templates/` (prebuilt XML/HTML templates consumed by helper scripts) and `scripts/` (Python tools like `template_tool.py`, `iconpark_tool.py`, `xml_text_overlap_lint.py` used at plan/generate time). Cross-cutting concerns — authentication, identity (`--as user` vs `--as bot`), scope handling, token mapping, permission-degradation flow — are centralized in `lark-shared/SKILL.md` and referenced from other skills with a `CRITICAL` directive to read it first. Skills also cross-reference each other for routing boundaries (e.g. `lark-base` defers file import to `lark-drive +import --type bitable`, auth/scope issues to `lark-shared`). There is no executable code here; the module is purely declarative knowledge consumed by the agent framework.
