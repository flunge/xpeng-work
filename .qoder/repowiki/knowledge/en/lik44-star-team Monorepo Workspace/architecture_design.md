Flat workspace with two independent project trees (`personal/meal/` and `team/`) plus a shared `.claude/` rules root. Both children are thin: they contain only Claude rules, reusable Python/Node scripts, and a Feishu token map; every piece of generated content (recipes, weekly reports, memory ledger) lives exclusively in Feishu, so the two projects never share filesystem state — they communicate indirectly through Feishu documents referenced by each child's `_feishu_map.json` / `feishu.yaml`. Scheduling is intentionally external: cron and trigger scripts run on the user's local machine and invoke the same `lark-cli` binary mounted at `/platform/.lark-cli`, which is the only runtime dependency shared across both children. Git submodules pull in `team/simworld` and `personal/resume` as nested workspaces without coupling them to the parent build.
