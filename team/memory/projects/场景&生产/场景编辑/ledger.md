## 技术架构

双模式共享IDM/MOBIL行为模型核心。用户手写JSON配置→Agent Service→逐帧计算轨迹→注入config_sim.yaml→上传OSS→扶摇渲染。改造后支持批量泛化/流程自动化/新旧兼容/可追溯，9大类问题诊断。

## 系统方案设计

（来源：场景编辑方案概要设计 LK8Qwjcx）

### 核心目标

**产出可大规模用于自动驾驶仿真的拟人化场景数据**，支持两种使用模式：

- **离线开环**：轨迹一次性生成，注入 yaml 后批量上传渲染（录制式仿真数据）
- **在线闭环**：渲染过程中按帧调用行为模型，根据自车与周边状况实时决策

两种模式共享同一行为模型核心（IDM/MOBIL + 参数采样器），区别仅在调用入口与时序。

### 技术架构（改造目标）

```
开环入口 (CLI/批处理) --> rollout(horizon) 一次性生成 N 帧轨迹 → yaml 注入
闭环入口 (Renderer Hook) --> step(ego_state) 每帧返回单帧动作 → 实时响应
                         共享：IDM/MOBIL + 参数采样器
```

当前实际架构（改造前）：用户手写 JSON / scenario_edit.csv → Agent Service（Trigger+Action）→ 逐帧计算轨迹 → 注入 config_sim.yaml → 打包 OSS → 提交扶摇渲染

### 支持的场景类型（当前）

5 种 JSON 模板：`cut_in_left`、`cut_in_right`、`cut_out_right`、`follow`、`dangerous_lc`

### 已识别问题（改造前）

| 层面 | 问题 | 影响 |
|------|------|------|
| 轨迹质量 | 匀速/匀加速，无跟车、无自适应 | 轨迹机械不拟人，仿真有效性低 |
| 多样性 | 固定模板，参数手动调 | 无法批量泛化 |
| 质量闭环缺失 | 无碰撞检测、无 TTC 筛选 | 无法保证产出质量 |
| 规模闭环缺失 | 无任务编排/重试/断点续跑，threading 受 GIL 限制 | 无法扩展 |
| 数据血缘缺失 | 仅时间戳区分，无 run_id 追溯 | 无法复现、无法对比 |
| 入口职责重叠 | 三个生成脚本并存，职责边界模糊 | 使用门槛高 |
