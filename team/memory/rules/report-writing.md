# 铁律 · 对外文档与汇报写作（report-writing）

> 单一事实源。写正式文档 / 周报双周报 / 述职材料的**内容与风格**规范（"写的时候"的规范；"发布前跑闸"见 publish-gate）。原 doc-rules + quality-rules 汇报叙事并入。

## 1. 正式文档规范（提交公司/管理层的材料）
- **不写晋升**：不主动建议/暗示某人晋升（晋升是 1:1/口头话题）。
- **人名用 @/`<cite>`**：结构性事实(离职/加入)用 @人名；不裸写人名、不评价人、不写细碎分工。判据：每个名字都用 @ 了吗？
- **精准 scope**：算法组=闭环仿真(SIL/HIL/生产链路/算法预研)，方向=3DGS重建/Fixer/Agent/WM预研/AVM鱼眼/场景泛化/CLIP-IQA。**不属于**：海外/机器人/RT/汇天(其他组业务线)、传感器仿真/交通流(不做)、大规模并行调度(平台组)。
- **写短**：表格单元格砍到要点、文字块只说判断结论、能删就删(HR 知道默认规则)。
- **信息边界**：群聊表扬≠正式素材、日会细节≠公司级内容；从 chat-log 提取的信息判断是否适合入正式文档。

## 2. 去 AI 腔（自查：这像我对老板汇报会说的话吗？像人话吗？）
禁用套话：赋能/抓手/闭环打通/已具备独立交付能力/首次尝试/可复用范式/放大团队效率；禁工整排比、禁 AI 替你下判断("比外招更能激励")、禁暗示晋升、禁群聊八卦当素材、禁解释默认规则。

## 3. 汇报叙事（述职/1v1，框架只做暗线）
- 让你参照某框架(价值观/干部七行/STAR)写汇报时，**框架隐含在内容里，绝不写成"对照X逐条看：客户至上：…"贴标签式列表**。
- 用 **STAR** 讲事：情境痛点→我的判断与行动→量化结果，别堆流水账/苦劳。
- 体现 **Ownership**：讲根因和你从源头做的取舍，不罗列动作。
- 成果**对齐业务价值**(训练提速2.6×→"客诉当天能复现")。
- 改写汇报**不动既定章节结构**，只润色语言。

## 4. 私下话术 vs 对外内容（剥离信息来源指代）
内部推演用的**来源指代**禁止出现在任何对外文档：

| 禁写 | 改写 |
|---|---|
| "老板双周会定调" / "高炳涛要求…" | 直接陈述结论/事项本身，不带来源 |
| "按老板三分法推进" | 说清优先级/方向本身，不提"谁定的" |

判据：这句在说"事情本身"还是"这事谁/在什么场合定的"？后者删或改中性。
- **review 私下话术要扫线上飞书文档**，不能拿本地产出物代替(本地无副本≠对外干净，如作战表 SBUYwm8L 纯线上)——列线上文档清单逐个 fetch 全文扫。
