## 三、会议机制（你定的框架细化）

> 背景：老板 6/28 23:33「七月以后精简各类会议时间、频率、参会人数，严查1小时日会」。
> 现状统计（6/11–6/26）：每天 2 场会、合计 ≈73min/天，下午例会平均 42min。

**新机制（7月起）：**

| 时间 | 会议 | 时长 | 内容 | 参会 |
|---|---|---|---|---|
| **周二 17:30** | 业务进展同步 | ≤30min | 场景生产/SIL/HIL/交付：对业务、跨组依赖、卡点 | 全组 |
| **周四 17:30** | 技术进展同步 | ≤30min | 渲染/Agent/预研/算法：技术攻坚、实验结论 | 全组（预研同学可选） |
| 周一/三/五 | 不开全员会 | 0 | 卡点小群异步解决；**日报每天照常更新** | — |
| **周五 15-16** | 实习生预研评审 | 1h | （已定） | 预研组+导师 |
| **周五 16-17** | 组内周会 | 1h | （已定）周复盘+下周计划 | 全组 |

**效果**：全员会从「每天1场×5」→「每周2场×30min + 周五2场」，会议时长降约 60%+。
**紧迫感不靠开会**：靠日报每天更新（公开承诺）+ 周五周会红黄绿考评。

### 主体：日报 Base（多维表格）

**为什么用 Base 不用文档表格**：满足你三个要求——①每行独立互不干扰 ②按 task 字段分组看目标 ③组员自己加行不需管理员。

**Base 字段设计：**

| 字段 | 类型 | 说明 |
|---|---|---|
| 日期 | 日期 | 默认今天，组员填 |
| 链路 | 单选 | 场景&生产 / SIL / HIL / Agents / 预研 |
| 关联月目标 | 单选/关联 | 挂到上面的某条 task（如「O1-KR3 渲染提速」） |
| 负责人 | 人员 | 自己 |
| 今日进展 | 多行文本 | 一句话进展 |
| 卡点/依赖 | 多行文本 | 需要谁支援、卡在哪 |
| 状态 | 单选 | 🟢正常 / 🟡风险 / 🔴滞后 |

**视图：**
- 视图1「按链路看」：按「链路」分组 → 每条线进展一目了然
- 视图2「按目标看」：按「关联月目标」分组 → 每个 task 下挂了哪些进展、是否滞后
- 视图3「今日」：筛选 = 今天 → 早会/周会直接看当天

**怎么用**：每个组员每天在 Base 里**新增一行**（=报一条进展），选好链路+目标+状态。无需管理员加行加列；多人同时填互不冲突。早会时切「今日」视图，按链路扫一遍即可。
