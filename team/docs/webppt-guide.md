# 网页版可编辑 PPT 制作指引

> 沉淀自 `ppt-slide-formatter` 项目（2026-06「小鹏自动驾驶仿真算法」技术分享网页版 PPT，课已讲完、项目已下线）。
> 保留这套方法论，供后续再做「整页位图 PPT → 可逐字编辑网页 PPT」时复用。

## 适用场景

拿到一套**整页位图**的 PPT（如 NotebookLM / 截图导出的每页一张 PNG），要把它重建成**纯前端、可逐字编辑**的网页版 PPT——正文由真实文本/HTML 元素承载而非整页图片，改文案无需重新出图。

## 技术选型（经验值，直接照搬）

| 维度 | 选型 | 理由 |
|------|------|------|
| 形态 | 纯前端静态站点（HTML/CSS/JS），无后端 | 网页可编辑、零依赖部署 |
| 构建 | **Vite（vanilla，无框架）** | `localhost:5173` 热更新；产物为静态文件 |
| 视图 | 单页 `index.html`，内含 N 个 `<section class="slide">` | 每页正文直接写 HTML，文本节点可独立定位编辑 |
| 样式 | 原生 CSS + CSS 变量（设计令牌 tokens.css） | 统一配色/字号/间距，保证多页视觉一致 |
| 交互 | 原生 JS | 只需翻页、键盘导航、全屏、进度条，逻辑轻量 |
| 字体 | 系统等宽（角标）+ 思源/系统无衬线（中文正文） | 中英混排不乱码 |
| Logo | 内联 SVG | 矢量清晰、统一复用；无官方素材则文字占位并标注 |

**不引入 React/Vue**：目标是「内容可手工编辑」，框架的组件化反而抬高编辑门槛。不做 .pptx/PDF 导出（超范围）。

## 核心设计原则

**内容即源码可编辑**：每页正文以语义化 HTML 直接书写（表格/列表/分栏/卡片/流程），文案修改只需在标记中定位文本节点，无需重新出图、无需运行生成工具。原图的文字、数值、层级必须**完整保留、不臆造**。

## 推荐目录结构

```
webppt/
├── index.html              # N 个 <section class="slide"> 主文档（正文在此编辑）
├── package.json            # vite 依赖与脚本 dev/build/preview
├── vite.config.js          # base、server.port=5173
├── public/assets/
│   ├── logo.svg            # 右上角统一 Logo
│   └── illustrations/      # 难以文本化的纯插画（线框图、流程底图）
├── src/styles/
│   ├── tokens.css          # 设计令牌：配色/字号/间距/边框
│   ├── base.css            # reset、字体、slide 容器、16:9 缩放
│   ├── hud.css             # 边框/角标/页眉页脚等通用装饰
│   └── components.css      # 标题页/对比表/卡片组/分支流程等布局组件
└── src/main.js             # 翻页/键盘/全屏/进度条/Logo 注入
```

每个 `<section class="slide" data-layout="...">` 结构统一：顶部 header 放角标+Logo 槽，中部 `slide-body` 放可编辑正文。

## 制作流程

1. **逐页抽取**：对照每张源图 `slide-NN.png`，把正文文字/数值/层级**逐字**抽成 HTML 元素，按内容形态归入布局原型（标题页/对比表/卡片/流程/引语）。
2. **建立设计令牌**：从源图提取配色/字号/间距，写进 `tokens.css`，作为全局视觉一致性基线。
3. **组件化布局**：把重复出现的版式（对比表、卡片组、分支流程）做成 `components.css` 里的 CSS 类，多页复用。
4. **校验**：对照源图逐页核对文案/数值无遗漏、无臆造；不清晰处以源图为准并标注。
5. **交付**：`vite build` 出静态文件，可直接部署或用飞书妙搭（lark-apps skill）发布成公网链接。

## 关键坑

- 中文正文务必指定思源/系统无衬线字体，避免字体缺失乱码。
- 整页位图里**难以文本化的纯插画**（线框车、示意底图）才保留为图片放 `illustrations/`；能文本化的一律转 HTML。
- 源图仅作**抽取内容的依据**，重建完成后源图无需随项目长期留存。
