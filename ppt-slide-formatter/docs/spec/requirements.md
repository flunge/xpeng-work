# 需求文档

## 功能概述

将一套 15 页「小鹏自动驾驶仿真算法」技术分享（深色科技 / HUD 风格，源自 NotebookLM 生成的整页图片 `refs/slide-01.png ~ slide-15.png`）转换为**可编辑的网页版 PPT**：每页内容由真实文本框 / HTML 元素承载（而非整页图片），并按用户反馈对首页、页眉 Logo、冗余装饰文字等做规范化处理。

## 用户故事

作为演示文稿的维护者（仿真算法负责人 Li Kun），我希望这套幻灯片是结构化、可逐字编辑的网页 PPT，并具备统一的品牌页眉和简洁的首页，以便我能随时修改文案、复用样式，而不必重新生成图片。

## 上下文与现状说明

- **源素材**：`refs/slide-01.png ~ slide-15.png`（已从 `归档.zip` 解压），为原始整页位图，内容不可编辑。
- **历史产物缺失**：上一会话疑似已构建过网页版 PPT，但引擎未能从磁盘恢复，工作区中**当前不存在** PPT 项目代码。后续阶段需以本需求 + 源图片为基准（重新）构建可编辑 PPT，而非假定已有实现。
- **风格基线**：深色背景、橙(#FF6A00 系)+ 青蓝(#00B4FF 系)高亮、等宽字体 HUD 边框与角标、中英混排（如「核心输入 (Core Input)」）。改造需保持该整体视觉风格。
- **重复装饰文字**：`DIGITAL TWIN TELEMETRY CONSOLE` 作为 HUD 角标在多页（如 slide-02 左上、slide-08 右下）反复出现。

## 功能需求

### FR-001: 首页改造为标准 PPT 标题页
- **类型**: 功能需求
- **优先级**: P0
- **描述**: When 渲染首页（slide-01）, the system shall 以标准标题页布局呈现主标题、副标题与主讲人信息，并去除碎块状标签内容。
- **验收标准**:
  - [ ] AC1.1: WHEN 展示首页 THE SYSTEM SHALL 显示主标题「打破工程困局」与副标题「如何在"不确定"的物理世界里，为自动驾驶大模型构建"确定"的运行闭环」
  - [ ] AC1.2: WHEN 展示首页 THE SYSTEM SHALL 显示主讲人信息为「仿真算法负责人：Li Kun」
  - [ ] AC1.3: THE SYSTEM SHALL 移除首页中碎块状的标签胶囊（`#VLA 2.0`、`#3DGS Recon`、`#Gen-Sim`、`#AI Agent`）及其他零散装饰块
  - [ ] AC1.4: 首页文本均为可编辑文本元素，主标题/副标题/主讲人可独立修改

### FR-002: 每页右上角添加小鹏（XPENG）Logo
- **类型**: 功能需求
- **优先级**: P1
- **描述**: When 渲染任意一页 slide, the system shall 在该页右上角固定位置展示小鹏 Logo。
- **验收标准**:
  - [ ] AC2.1: THE SYSTEM SHALL 在全部 15 页 slide 的右上角统一位置展示 XPENG Logo
  - [ ] AC2.2: THE SYSTEM SHALL 保证 Logo 大小、边距在各页一致，不遮挡正文内容
  - [ ] AC2.3: IF 缺少官方 Logo 素材 THEN 使用清晰可辨的 XPENG 文字/标识占位 并在交付说明中标注待替换

### FR-003: 删除 DIGITAL TWIN TELEMETRY CONSOLE 装饰文字
- **类型**: 功能需求
- **优先级**: P1
- **描述**: When 渲染各页 slide, the system shall 不再出现「DIGITAL TWIN TELEMETRY CONSOLE」文字框。
- **验收标准**:
  - [ ] AC3.1: THE SYSTEM SHALL 删除所有页面中出现的「DIGITAL TWIN TELEMETRY CONSOLE」文字（含 HUD 角标处）
  - [ ] AC3.2: WHEN 删除该文字后 THE SYSTEM SHALL 保持原有 HUD 边框/布局不塌陷、视觉协调

### FR-004: 将图片型内容抽离为可编辑文本/元素
- **类型**: 功能需求
- **优先级**: P0
- **描述**: When 某页内容仍以整页（或大块）原始图片呈现, the system shall 将其文字、列表、对比表、流程图等内容抽离为结构化的可编辑文本框 / HTML 元素。
- **验收标准**:
  - [ ] AC4.1: THE SYSTEM SHALL 使全部 15 页均不使用整页位图承载正文，正文文字以可编辑文本元素呈现
  - [ ] AC4.2: WHEN 原图含结构化内容（如 slide-02 的「传统模块化 vs VLA 2.0」对比表、slide-08 的「NPC 变轨 / 资产替换 / 无限繁衍」分支说明）THE SYSTEM SHALL 以对应的表格/列表/分栏结构还原，文字可逐项编辑
  - [ ] AC4.3: THE SYSTEM SHALL 完整保留原图中的中英文文案、数值（如「<80ms」「<10ms」）与层级关系，不遗漏不臆造
  - [ ] AC4.4: IF 某些纯插画/示意图（如首页线框车、流程连线图）难以用文本还原 THEN 可保留为独立图片元素，但其上的说明性文字须抽离为可编辑文本

## 技术约束
- 保持现有深色科技 / HUD 视觉风格（配色、等宽字体、边框角标），改造不得破坏整体观感。
- 幻灯片总数保持 15 页，页序与原 `slide-01 ~ slide-15` 一致。
- 产出物须为网页技术栈（HTML/CSS/JS）实现的可编辑 PPT，便于直接修改文案。
- 中文为主、中英混排需正确渲染，避免字体缺失导致的乱码。

## API 依赖
- 无后端 API 依赖；为纯前端静态演示项目。Logo 素材为静态资源依赖（需提供或占位）。

## 非功能需求

### NFR-001: 可编辑性
- AC1.1: 每页所有正文文字均可在源码中以独立文本元素定位并修改，无需重新出图。

### NFR-002: 视觉一致性
- AC2.1: 15 页在配色、字体、页眉 Logo 位置、HUD 边框样式上保持统一规范。

## 范围说明（本轮不实现）
- 不新增超出原 15 页的内容页，不改写技术文案的实质含义（仅做结构化与上述 4 项规范化处理）。
- 不实现导出 .pptx / PDF 等格式转换（如需另立 Feature）。
