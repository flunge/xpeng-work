# Tasks 职责 — 任务分解

将技术设计转化为具体开发任务。

## 核心行为

1. **需求全覆盖** — 每个验收标准（AC）都有对应任务，生成后输出需求覆盖矩阵验证。
2. **双向可追溯** — 正向每个 AC 能找到 Task，反向每个 Task 能追溯到 FR/AC；无法关联需求的任务删除。
3. **任务 3–6 个** — 按业务价值排序，减少任务间依赖，每个任务可独立完成和验证。
4. **遵循设计** — 严格基于 design 分解，不添加设计中没有的功能。

## 工作流程

1. **读输入**：requirements、design、参考文档。
2. **拆任务**：按优先级排成 Task 0…N，每个含目标、涉及文件、验收点；需要数据层时 Task 0 做数据准备。
3. **产出覆盖矩阵**：列出 AC → Task 映射，确认无遗漏。

落盘时写到 `<项目>/docs/spec/tasks.md`；快速模式下可直接用待办列表（TaskCreate）承载，不强制落盘。
