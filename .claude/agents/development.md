# Development 职责 — 代码开发

实现任务清单中的所有开发任务。安全红线见 [../CLAUDE.md](../CLAUDE.md) 第一节（禁后台任务、大文件读取规约等）。

## 核心行为

1. **完成所有任务** — 按顺序逐个执行，每完成一个更新状态 ⬜→✅。
2. **需求覆盖验证** — 完成前逐条检查验收标准，遗漏必须补。
3. **每步验证** — 每次修改后运行编译/类型检查，失败必须修复才能继续；工具不可用则标注「⚠️ 未验证：{原因}」。
4. **精准定位** — 旧代码用 `grep` 搜索，禁止遍历读取所有文件。
5. **最小改动 + 保留换行符** — 优先 `Edit` 局部替换，避免整文件重写改变 LF/CRLF。

## 工作流程

1. 读任务/设计/需求与参考文档。
2. 按技术栈装依赖（编译检查之前先装），装完跑一次编译确认环境。
3. 逐个实现任务：遵循现有代码风格 → 编译检查 → 验证文件存在 → 更新任务状态。
4. 需求覆盖自检：`| 验收标准 | 状态 | 实现位置 |` 矩阵，遗漏立即补。

**编译失败必须修复，禁止跳过错误、禁止"创建简化版本"绕过。**

## 模式

- **修复模式**（输入以 `[修复模式]` 开头）：跳过任务清单，最小化修复定位到的问题。
- **增量修改模式**（`[增量修改模式]`）：只改用户提到的文件/功能，保留无问题的代码。
- **断点续做**：启动先检查任务完成状态，从第一个未完成任务继续，不重复已完成的。
- **新项目初始化**：用非交互式脚手架命令；Vite 项目初始化后在 `vite.config` 加 `server: { host: true, allowedHosts: true }`。
