# 🍳 家庭食谱系统

一家四口（夫妻 + 4岁/6岁男宝）的自动化食谱管理。

## 使用说明

### 日常使用
每晚 **18:00** 自动收到飞书通知，告诉你**明天**的食谱、采购清单和前一晚可以做的准备。

### 今晚就开始
今晚看一下飞书消息，按照"前一晚准备"的步骤操作即可。

### 每月更新
**每月最后一天 20:00** 自动生成下月食谱，每天不重样。

## 目录说明

```
meal/
├── config/               # 配置文件
│   ├── family.yaml       # 家庭信息（人数/年龄/偏好/厨具）
│   ├── holidays-2026.yaml # 全年法定节假日
│   └── webhook.yaml      # 飞书 Webhook URL
├── recipes/              # 食谱库（YAML 格式）
│   ├── breakfast/        # 早餐食谱
│   ├── lunch/            # 午餐食谱（节假日/周末用）
│   ├── dinner/           # 晚餐食谱（节假日/周末用）
│   └── side/             # 加菜池（素菜/凉菜/汤，节假日/周末凑齐3菜）
├── plans/                # 生成的月度计划
│   ├── 2026-06.md        # 本月概览
│   └── daily/            # 每日详细食谱卡片
├── scripts/              # 自动化脚本
│   ├── generate_month.py # 月度计划生成器
│   ├── notify_daily.py   # 飞书通知发送
│   ├── run_daily.sh      # 每日通知包装脚本
│   └── generate_next_month.sh # 下月计划包装脚本
├── setup.sh              # 环境引导/重启恢复脚本（装依赖+crontab）
└── notifications/        # 通知日志
```

## 部署 / 重启恢复

本系统部署在 Agent Pod。工程文件（`/workspace/meal`）和飞书授权（`/platform/.lark-cli`）在持久盘上，Pod 重启不丢；但 PyYAML、crontab、cron 守护进程在容器临时层，重启后需恢复。

**Pod 重启后跑一次即可：**

```bash
bash /workspace/meal/setup.sh
```

## 手动运行

```bash
cd /workspace/meal

# 生成指定月份的食谱计划
python3 scripts/generate_month.py --year 2026 --month 7

# 手动发送明日通知
python3 scripts/notify_daily.py
```

## 如何添加新食谱

在 `recipes/` 对应目录下创建 `.yaml` 文件，按以下格式：

```yaml
title: "菜名"
type: breakfast  # 或 lunch、dinner
difficulty: easy
total_time: "用时描述"
servings: 4
tools: [工具1, 工具2]
ingredients:
  菜品1:
    - name: 食材名
      amount: 分量
      note: 备注（可选）
night_prep:
  - "前一晚可以做的操作"
morning_steps:  # 早餐用
  - "早上操作步骤"
notes: "注意事项"
```
